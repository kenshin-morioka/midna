# midna プロジェクトでの作業ルール

`~/.claude/CLAUDE.md` に上乗せする、このリポジトリ固有のお作法。**短く保つ**（要点だけ。背景や言い換え集は memory / ADR / 雛形ファイル側に書く）。

## 指摘されたら同じターンで再発防止まで

1. このファイルに該当ルールを 1〜2 行で追記
2. リポジトリを `grep` 等で走査し、同じ違反が残っていないか点検
3. 残っていれば直して同じコミット群に入れる

## 設計判断は ADR

- 方針や引き換えを伴う判断は `docs/adr/NNNN-kebab.md` に残す
- design doc には判断詳細を書かず ADR へのリンクだけ置く
- 実装の細かい選択は ADR にしない

## ADR の書き方

- 構成は **結論 → 背景 → 理由 → 影響 → 他に検討した案**（雛形は `docs/adr/README.md`）
- 本人視点で書く。「ユーザーは○○と明示」のような第三者目線は禁止
- 「他に検討した案」は本気で比較した案だけ。勘違い候補を後付けで足さない
- 「良い点」と「引き換え / 困る点」を両方書く

## 文章は自然な日本語で

- ADR / design doc / README / CLAUDE.md / PR 本文 / コメントは日本語
- 普段の会話に出てこないカタカナ語（ロックイン、トレードオフ、エコシステム、フロンティア、ペイン、YAGNI、オーバーエンジニアリング 等）は使わず言い換える
- 技術固有名（Rust, tokio, trait, HTTP, ADR 等）はそのまま OK

## 推測で断定しない

- 弱いヒントだけで言語・スタックを決めない。確定的な手がかりがなければ確認する
- 推測で進めるなら「推測です」と明示

## 終わるときの手順

コミット前に動作確認（`cargo build` / `cargo test`、必要なら `cargo clippy`）→ push → 必要なら PR 作成。

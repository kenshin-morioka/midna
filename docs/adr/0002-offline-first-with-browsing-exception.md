# 0002 - 推論はオフライン、Web を見に行く道具だけ通信を許可

- 状態: 採択
- 日付: 2026-05-18
- 決定者: kenshin-morioka

## 結論

- **推論は完全にオフラインで動かす。** LLM への問い合わせは手元のランタイム（Ollama など）に閉じ、外部の API は呼ばない。
- **例外として、Web に情報を取りに行く道具だけは外部通信を許可する。** Web 検索や Web からの取得に限り、必要に応じて通信してよい。
- **初回のセットアップに必要な通信は別扱い。** ランタイムのインストール、モデルファイルの取得、ライブラリの取得などはネット前提でよい。これは「実行中の通信」とは切り分けて考える。

## 背景

design doc v0.1 の "Core Philosophy / Local-first" は「手元優先・クラウドはおまけ」と書いてあるが、ユーザーは「wifi なしで動く AI エージェントが欲しい」と明示している。"Local-first" の意味をもう一段はっきりさせ、どこまで外部通信を許すかを決める必要がある。

判断の軸:

- プライバシー（自分用の道具なので、会話の中身を外に送らない）
- 通信なしでも基本動作が成立すること
- 最新情報を扱う用途（Web からの情報取得は本来ネット前提）
- 初回セットアップの現実性（モデル取得などはネットが要る）

## 理由

- 自分専用の道具なので、会話そのものは手元で完結するのが望ましい。
- 一方で「今の情報を聞きたい」のようにネット前提の用途を全部塞ぐと使い物にならないので、穴をあける範囲を明示的に限る。
- 初回セットアップは「実行中の通信」とは別の話として扱うのが自然。ビルド時の通信まで禁じると、そもそも準備ができない。

## 影響

- 良い点
  - 通信なしで会話ができることを、はっきり保証できる（飛行機や移動中など、ネットが安定しない場面でも使える）
  - 会話の記録が外に残らない（プライバシーを守れる）
  - 外部 API の費用が発生しない
- 引き換え / 困る点
  - 初回のモデル取得とランタイムの導入はネットが必要（手元にネットがない環境では事前に用意しておく必要がある）
  - クラウドの最新の高性能モデルを使いたい場合は、別途明示的に切り替える仕組みを足す必要がある
  - Web を見に行く道具を作るときは、許可する通信先のルールを別途決める必要がある（権限の仕組みで扱う）

## 他に検討した案

- **クラウド API を主役にする** — Anthropic や OpenAI を一級扱いにする案。品質と速さは有利だが、「手元で動かしたい」という意向と相反するため不採用。
- **完全オフラインで、Web を見に行く道具も禁止** — もっとも厳しい設計。だが最新情報を取れないのは実用上の損失が大きいため不採用。
- **個別の道具ごとに通信ポリシーを後で決める** — 方針がないまま実装が進むと一貫性が崩れるため、v0 の時点で方針を残すことにした。
